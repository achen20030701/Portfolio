## 6. 面试准备

### 6.1 项目介绍话术

**AI 智能问答助手（1分钟）**：
> "我开发了一个基于云函数的 AI 智能问答助手。技术栈是 Node.js + 腾讯云 CloudBase + DeepSeek API。核心特点是 Serverless 架构，无需管理服务器，按调用次数计费。支持多轮对话，自动保存历史记录。已部署上线，可以在线体验。"

**RAG 文档问答系统（2分钟）**：
> "我开发了一个 RAG 文档问答系统，支持用户上传文档，然后基于文档内容进行语义问答。技术栈是 Python + FastAPI + LangChain + Chroma + DeepSeek。核心流程是：文档上传 → 文本分块 → 向量化 → 存入向量数据库 → 检索相似文档 → 生成回答。这个项目展示了我对 RAG 架构、Embedding、向量数据库和 Prompt Engineering 的理解。"

### 6.2 技术深度问题准备

**Q: 为什么选择 RAG 而不是微调？**
> "RAG 有几个优势：1）知识可更新，不需要重新训练模型；2）回答可追溯，能提供来源；3）成本低，不需要 GPU 训练。对于企业知识库场景，RAG 是更合适的选择。"

**Q: 如何优化 RAG 的检索效果？**
> "可以从几个方面优化：1）分块策略，调整 chunk_size 和 chunk_overlap；2）Embedding 模型，选择更适合中文的模型；3）混合检索，结合向量检索和关键词检索；4）Rerank 重排序，对检索结果进行重排。"

**Q: 云函数的冷启动问题如何解决？**
> "冷启动是 Serverless 的常见问题。解决方案包括：1）保持函数温热，定时触发；2）使用预留实例；3）减少依赖，加快加载速度；4）选择轻量运行时，如 Node.js 比 Java 冷启动更快。"

### 6.3 行为面试问题准备

**Q: 遇到过什么技术难点？如何解决的？**
> "在 RAG 项目中，遇到检索效果差的问题。通过分析发现是分块太大导致噪音多。解决方案是减小 chunk_size 到 500 字符，并增加 chunk_overlap 到 50 字符，保持上下文连贯。同时优化了 Prompt 模板，让 LLM 更严格地基于检索内容回答。"

**Q: 如何保证代码质量？**
> "我遵循几个原则：1）代码结构清晰，模块化设计；2）有完整的文档，包括 README 和技术解析；3）有错误处理和日志记录；4）可以在线体验，验证功能正确性。"
